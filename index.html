Differences between Front End and Back End

>First of all, a user is capable of looking and interact with the front end, while they are not capable of manipulating the back end directly.Also, front end has to do more with designing the arrangement of components so it looks friendly to the client, and back end works the arrangement of data and efficiency of tasks for keeping order in the server. Finally,developping one of these parts uses different programming languages, for example Java or Python for back end and HTML and CSS for front end.

Who is the end user?

The client is not always the user. The end user is the person in front of the computer that will log into the website or plattform and use the functionalities of the server.In many cases, the user does not need to have any direct relation with the client or the person who built the website.

Who is the Client?

The client is the company that requested the service of building a website and receives the goods in the end. They are the ones with which the software creator establish a business relationship since they will indicate the requirements and they are who the designer will have constant communication with.

CDN Architecture

CDN stands for Content Delivery Network. It basically functions as form of getting the best performance of a website by stratigecally placing servers across the world so that the users connect to a server close to them. Users can request content and get or receive the content from a storage cloud upon request.

History of HTML and CSS

HTML was created in 1990 as a language for creating documents on the world wide web. It uses a variety of tags solely to define the structure of a written electronic document. However, in 1994, HTML 2.0 appeared with new tags for adding images, but many browsers didn't support images. Some time later, HTML added tables in their third version, which helped to give a better layout to a website since you could arrange images and text in a more organized way. With HTML 3.2 the web designer role was created. You were able to set fonts, colors, backgrounds and more.In spite of all the advances, websites couldn't display in the same form in all the browsers due to propieraty tags. In 1996, it was decided that content and display should be separated entinties, that cause the creation of CSS (Cascading Style Sheets), which brought many advantages to styling a website. CSS also developped new versions to give designers more freedom at creating the layout of the website. Nowadays, browsers display websites in basically the same way, and CSS is now focusing on improving user experience implementing aural stylesheets.
